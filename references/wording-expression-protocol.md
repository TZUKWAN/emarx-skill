# EMARX 遣词造句与表述协议

起草、修改或审稿中文学理论文句级表达时使用本文件。

## 核心问题

很多弱稿并非每句都有语法错误，而是句子起点不对：先下判断再点名对象、先否定再建立语境、用抽象名词掩盖空关系。

既有语言蒸馏和失败稿审查还暴露另一种问题：句子不是在论述对象，而是在解释论文自身。“本文认为”“本文的核心观点是”“本文将”“文章认为”“有研究指出”会让 EMARX 听起来像助手在解释作业，或在正文里插入了一段综述。

好的学术表达应让读者知道：

- 讨论的是什么对象；
- 澄清的是什么关系；
- 有什么证据或理论依据；
- 段落推进了什么判断。

## 段落开头

段首应给读者定位。可以从以下位置进入：

- **对象**：点名具体对象、文本、政策、案例、概念或关系；
- **学术位置**：点名学术争论或理论问题；
- **材料**：以来源、案例、数据或政策事实开头；
- **过渡**：连接上一段与下一段分析运动；
- **概念边界**：澄清本文如何使用某个概念。

禁止以裸否定开头。

弱段首：

- 不是……
- 并非……
- 不能……
- 不应……
- 不要……
- 没有……
- 无需……
- 并不……
- 绝非……

这些形式在对象建立后可用于概念边界，但在读者尚不知纠正对象时显得无力。

与其以否定开头，不如先点名对象或关系：

```text
弱：不能把生成式人工智能简单理解为传播工具。
强：生成式人工智能进入文化传播后，内容被组织和调用的方式随之改变。将其理解为普通传播工具，会遮蔽模型、平台和语料共同塑造意义的过程。
```

第二版仍包含纠正，但纠正跟在对象和理由之后。

## 表述失灵类型

以下表述问题有共同根源：在对象和关系尚未清楚之前，就要求读者接受转折、判断或抽象。

### 1. 空泛起笔

弱起笔：

- 在……背景下
- 随着……的发展
- 当前
- 近年来
- 从……来看
- 在……语境下

这些起笔并非绝对错误，但常常拖延真实对象。仅当句子迅速进入具体关系时才可使用。

### 2. 悬空指代

弱起笔：

- 这种
- 这一
- 上述
- 这说明
- 这意味着

这些词需要就近指代。不要用它开启新段落，除非上一段已清楚准备对象且新段立即点名承接内容。

### 3. 过早判断

弱起笔：

- 关键在于……
- 核心是……
- 根本上……
- 本质上……
- 必须……

这类起笔常在分析前给出结论。应在段落已说明判断为何必要之后再使用。

### 4. 概念堆叠

弱句把几个抽象术语堆在一起：

```text
主体性、解释关系、治理秩序和价值协同构成了智能传播的结构基础。
```

这听起来学术，但隐藏了关系。应改写为：哪个术语通过什么机制作用于哪个对象。

### 5. 伪过渡

过渡本身在语料中常见，并不坏。坏的是它让文章看起来连贯而推理并未移动：

- “因此”用于并未推理之处；
- “由此可见”用于仅重复陈述之处；
- “与此同时”作为松散连接；
- “值得注意的是”用于下文并无特别之处；
- “进一步看”用于并无进一步分析之处。

仅当过渡词点名真实运动时才使用：概念到材料、材料到机制、机制到判断、文献局限到本文入口。

### 6. 只有断言的段落

段落若每句都在断言而没有证明、区分、举例或解释，就是失败的。段落需要论证动作，而非只有话题。

### 7. 口号化结尾

避免仅以听起来正面收束：

- 具有重要意义
- 提供有力支撑
- 形成强大合力
- 开辟新路径
- 提升传播效能

结尾应落在已澄清的内容上，而不是问题有多重要。

### 8. 说明书腔

路径和治理段落容易变成工作说明。注意反复使用“应”“应当”“需要”“必须”“建议”等情态动词。

这些词并非禁用，问题在于段落只剩对行动者发号施令，而不再解释该行动如何从论文分析中推出。

好的路径写作应这样运动：

```text
已诊断的机制 -> 必须改变的条件 -> 行动者与行动 -> 该行动如何回应机制
```

避免：

- 连续以“应……”开头的句子；
- 由机构任务组成的段落；
- 没有理论回归的行动清单；
- 把路径段当作实施手册。

## 句子方法

段落松散时，使用默认运动：

```text
对象锚定 -> 关系澄清 -> 证据/材料 -> 解释性判断
```

不要使用这种运动：

```text
论文自我说明 -> 论断宣告 -> 来源罗列 -> 建议
```

句子功能示例：

- 对象锚定：点名讨论对象；
- 关系澄清：解释 A 如何影响 B、属于 B、区别于 B 或生成 B；
- 证据/材料：引入文献、政策、案例、文本、数据或历史语境；
- 解释性判断：说明该关系对论文论证意味着什么。

避免段落只包含：

- 论断 -> 论断 -> 论断；
- 概念 -> 概念 -> 概念；
- 政策用语 -> 口号 -> 建议；
- 否定 -> 纠正 -> 抽象结论；
- “本文认为” -> “有研究指出” -> “因此应当”。

## 无主语摘要与正文语态

摘要通常不应出现可见作者主语，应以第三人称压缩学术语态直接描述对象。

禁用：

- 本文认为……
- 本文的核心观点是……
- 本文从……展开分析
- 笔者认为……
- 本研究旨在……
- 文章认为……

更好运动：

```text
研究对象 -> 问题关系 -> 机制/逻辑 -> 价值或路径
```

正文同样适用此规则。若句子以解释论文要做什么开头，请改写为关于对象的句子。

## 文献进入：消化而非综述

避免把文献作为 detached 综述引入正文：

- 有研究指出……
- 已有研究认为……
- 相关研究指出……
- 学者认为……

需要文献时，先把它消化为四种功能之一：

- 概念来源；
- 机制来源；
- 问题来源；
- 限制来源。

句子仍应属于论文自己的论证。引用标记可跟在句后，但文字读起来不能像在正文里贴了一段文献综述。

## 否定纪律

否定对概念边界工作有用，但不能成为行文发动机。

仅在以下情况使用否定：

- 存在真实误解需要纠正；
- 正面论断紧随其后；
- 句子点名了被纠正的对象；
- 论文有证据或推理支持该纠正。

避免：

- 段首以否定开头；
- 反复使用“不是……而是……”；
- 仅为制造戏剧对照的否定；
- 纠正文中无人真正持有的观点。

更好运动：

```text
先说明对象的实际运行方式，再指出旧理解的不足，最后给出本文判断。
```

## 动词选择

优先使用能显示智识动作的动词：

- 区分
- 澄清
- 解释
- 揭示
- 勾连
- 限定
- 转化
- 支撑
- 削弱
- 拓展

中文表达中优先使用直接动词：

- “批判……”优于“对……进行批判”
- “解释……”优于“对……作出解释”
- “限制……”优于“形成对……的限制”
- “支撑……”优于“为……提供支撑”

除非句子同时点名对象、机制和条件，否则慎用“推动”“促进”“加强”“优化”“提升”“赋能”“打造”“构建”。

## 主宾清晰

每个重要句子都应有清晰主语和宾语。

弱信号：

- 主语是“这一问题”“这种情况”“这一逻辑”等抽象容器且缺少就近指代；
- 动词过于笼统，无法显示关系；
- 宾语缺失，读者看不出什么发生了变化；
- 句尾以“重要意义”“现实价值”“内在要求”收束，却不指明价值对象。

修复时点名：

```text
行动者 / 机制 / 对象 / 后果
```

## 过渡

过渡应解释下一段为何必要。

好的过渡：

- 从概念到材料；
- 从材料到机制；
- 从机制到治理含义；
- 从文献局限到本文入口。

语料中常见“因此”“由此可见”“与此同时”“进一步看”“在此基础上”“换言之”“这意味着”“然而”“但是”“另一方面”。这些在点名真实运动时可以使用，仅在伪装非运动为运动时变弱。

避免：

- 没有推理时用“因此”；
- 仅重复陈述时用“由此可见”；
- 把“与此同时”当松散连接；
- 下文并不特别时用“值得注意的是”。

## 标题与正文区分

正式一级、二级标题仍然需要。正文段落不应变成隐藏提纲。

正文段落内部尽量少用“一方面……另一方面……”“首先……其次……”“第一……第二……”等显性脚手架。需要呈现真实步骤时，应优先让对象关系和过渡句承担组织功能；若使用序列词，必须确认它没有把段落变成口号清单。

还应避免：

- 冒号小标题；
- 仅罗列概念的分号堆叠；
- 结尾像政策要点的段落；
- “问题在于：”“关键在于：”“核心观点是：”等冒号提示-回答结构。

## 修改清单

修改时检查：

```text
裸否定段首：
无对象锚定的段落：
只有抽象概念的段落：
主语不清的句子：
泛化动词：
无支撑的过渡：
只有断言的段落：
口号化结尾：
可见作者主语：
论文元话语：
综述插入套式：
冒号小标题：
```

若段首不好，不要只换词。应从对象、关系、材料、判断重新搭建段落。
