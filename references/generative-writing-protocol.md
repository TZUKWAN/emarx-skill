# EMARX 正向论文语言生成协议

本协议是 EMARX v6.9 的核心生成规则。它规定的不是“写完后如何检测坏稿”，而是“落笔之前和落笔之时，论文语言应当如何生成”。审稿、审计和脚本只作为最后兜底；真正的质量控制来自生成前的协议、段落级大纲和逐段生成时的自我校准。

## 总则：先写对，再审好

EMARX 过去的问题不是检测项太少，而是默认生成机制偏离了真实中文学理论文的语言运动。v6.9 把规则前移：

- 摘要必须在生成前确定无主语报道式结构，而不是写完后删除“本文”。
- 正文段落必须在生成前确定对象、关系、材料、判断和句式走向，而不是写完后用脚本抓错。
- 文献必须在进入句子的瞬间被消化为概念来源、机制来源、问题来源或限制来源，而不是先写“有研究指出”再回头修改。
- 路径段必须从已论证的机制自然推出，而不是列出一组“应当加强”的行动清单。

生成协议优先于审计协议。写作时若发现难以遵守本协议，应退回大纲和诊断阶段，而不是寄希望于后期修改。

## 摘要生成协议

### 默认结构

摘要采用无作者主语、第三人称、面向对象的报道式表达，按以下运动生成：

```text
对象进入 -> 问题关系 -> 机制判断 -> 风险或价值 -> 路径指向
```

### 禁用结构

不要按以下运动生成摘要：

```text
本文说明 -> 本文认为 -> 本文从几个方面分析 -> 本文提出路径
```

### 每句功能

1. **对象进入**：第一句直接呈现研究对象，可以是现象、概念、政策、技术或事件。不要先解释“本文研究什么”。
2. **问题关系**：第二句建立对象内部的张力、矛盾或转换关系，使读者意识到这不是常识描述。
3. **机制判断**：第三句给出核心判断，说明是什么机制或逻辑造成了上述关系。
4. **风险或价值**：第四句指出该机制带来的理论风险、实践风险或价值指向。
5. **路径指向**：最后一句收束到论文的落脚点，可以是理论判断、治理方向或进一步的问题。

### 正例

> 生成式人工智能进入国际传播后，文化传播的内容生产、语义调用和跨语境适配方式发生了结构性变化。模型训练数据、平台分发逻辑与用户Prompt共同塑造了新的意义生成环境，使中华文化国际传播既获得多语言、多模态的扩散能力，也面临文化语义被稀释、解释权被让渡的风险。问题在于，技术效率的提升并不自动转化为文化意义的深度抵达，算法对语料概率的优化可能遮蔽文化理解所需的语境厚度。因此，中华文化国际传播需要从“内容输出”转向“意义协商”，在语料治理、平台反馈机制和跨文化解释主体建设之间形成协同。

### 反例

> 本文旨在研究生成式人工智能语境下中华文化国际传播的机遇、挑战与路径。本文认为，生成式人工智能为中华文化国际传播带来了新机遇，也带来了新挑战。本文从传播机制、风险生成和治理路径三个方面进行分析，并提出了相应的对策建议。

反例的问题：作者主语化、自我说明、结构暴露、缺少机制判断。

## 正文段落生成协议

### 生成前的后台条件

每段落笔前，先在内部确认以下五项，但它们只能作为生成条件，不能写入正文：

```text
本段对象：
本段关系：
本段材料：
本段判断：
本段如何承接上一段：
本段如何引出下一段：
```

如果任何一项无法清晰回答，不要开始写这一段。返回大纲或重新诊断。

### 段落运动

正文段落按以下运动生成：

```text
对象锚定 -> 关系展开 -> 材料或文献进入 -> 机制解释 -> 判断落点
```

禁止按以下运动生成：

```text
本文认为 -> 有研究指出 -> 因此应该
```

### 各环节说明

1. **对象锚定**：第一句给出具体对象。可以是政策文本、技术机制、案例、概念、历史事件、数据或平台现象。不要悬空指代，不要以否定或抽象判断开头。
2. **关系展开**：说明对象内部或对象之间的作用关系。谁影响谁、谁转化谁、谁限制谁、谁被遮蔽。
3. **材料或文献进入**：在关系展开后引入材料或文献。文献必须先消化：它提供了什么概念、机制、问题或限制？引用以 `[X]` 形式紧随使用它的句子，不堆在段落末尾。
4. **机制解释**：解释关系为何成立，说明因果链条、结构条件或理论逻辑。
5. **判断落点**：段落以具体判断收束，回到论文的核心问题，并为下一段留下过渡。

### 正例

> 生成式人工智能的跨语言生成能力首先改变了文化内容的转译方式。传统翻译以人工或规则系统为主，目标是在两种语言之间建立对应关系；大模型则基于海量语料的概率分布，直接生成符合目标语境表达习惯的文本[1]。这种转换提高了传播效率，但也使原文的文化语义让位于目标语境的统计偏好。当“春节”被模型处理为类似“中国新年”的通用表述时，节俗背后的时间观念和家庭伦理便被压缩为可快速消费的符号。由此，技术便利性并未自动等同于文化理解，反而可能在扩散中削弱意义的厚度。

### 反例

> 本文认为，生成式人工智能对中华文化国际传播有重要影响。有研究指出，人工智能可以提升翻译效率[1]。因此，我们应该加强技术应用，推动中华文化更好地走出去。具有重要意义。

反例的问题：自我说明、综述插入、缺少对象锚定、材料未消化、判断口号化。

## 文献消化式进入方法

文献不是论文的装饰，而是论证的养分。进入正文前，先回答：

- 如果文献提供概念，就把概念边界写进句子。
- 如果文献提供机制，就把机制关系写进论证。
- 如果文献提供问题，就把问题转化为本文的切入点。
- 如果文献提供不足，就把不足转化为本文的推进空间。

### 操作规则

1. **不让作者名字和“指出/认为”成为句子主体**。句子的主语应是对象、机制或关系，而不是“某学者”。
2. **引用紧随使用**。引用放在具体使用该文献的句子之后，以 `[X]` 形式出现。
3. **一个文献只引一次，一个作者只引一次**。在引用规划阶段就确定每篇文献的论证功能，不要临时补引。
4. **文献服务本文判断**。如果删除某条引用后段落的判断仍然成立，说明该引用只是装饰，应删除或替换。

### 正例

> 文化记忆研究把记忆理解为社会框架下的重构过程，而非对过去的简单保存[1]。这意味着，中华文化国际传播中的“传统”并非静态遗产，而是在当代媒介环境中被不断选择、编码和再阐释的对象。

### 反例

> 有研究指出，文化记忆是社会框架下的重构过程[1]。相关研究认为，传统在传播中会被不断阐释[2]。

反例的问题：文献成为主体，未进入论证，综述堆砌。

## 路径段生成协议

路径段不是行动手册，而是论证的延伸。它必须从已建立的机制判断中自然推出。

### 生成结构

```text
已诊断的机制 -> 需要改变的条件 -> 行动者及其行动 -> 该行动如何回应机制 -> 理论收束
```

### 禁用结构

不要写成：

```text
问题 -> 因此应当 -> 一、加强... 二、完善... 三、优化...
```

### 操作规则

1. **每个路径都对应一个已论证的机制或风险**。不能先写“应当加强语料建设”，再倒补理由。
2. **行动要有主语和对象**。谁、对什么、做什么、达到什么效果。
3. **避免连续使用“应”“应当”“需要”“必须”开头的句子**。路径段可以有规范性判断，但不能只剩下规范判断。
4. **路径段末尾要回到理论判断**。说明这些行动如何改变对象关系，而不仅是完成任务。

### 正例

> 要稳固中华文化国际传播的语义厚度，就需要在模型训练的前端介入语料治理。语料不仅决定生成内容的语言形式，也决定了文化概念被调用时的默认语境。当高质量、语境完整的中文文化语料成为训练基础时，模型输出才更可能在跨语言转换中保留意义的层次。这一治理方向并不意味着回到封闭的文化保护，而是通过提升训练数据的文化密度，使技术效率与文化理解形成正向循环。

### 反例

> 针对上述问题，我们应该：一、加强技术研发；二、完善语料建设；三、优化传播平台；四、培养专业人才。只有这样，才能推动中华文化国际传播高质量发展。

## 结论生成协议

结论不是结构复述，而是理论判断的升华。

### 生成结构

```text
回到核心判断 -> 提升抽象层次 -> 指出边界或余论 -> 收束全文
```

### 操作规则

1. **不要重复“本文通过……分析了……”**。直接给出判断。
2. **把具体判断上升为机制或理论命题**。说明本文的分析对理解更大范围的问题有何启发。
3. **可以指出一个限制或反例**，以增强判断的分寸感。
4. **最后一句落点明确**，不要泛泛强调“具有重要意义”。

### 正例

> 生成式人工智能语境下的中华文化国际传播，本质上是一场关于意义解释权的竞争。技术扩散放大了文化传播的规模，却也在概率优化中压缩了文化理解的深度。论文的分析表明，只有在语料治理、平台反馈和解释主体建设之间形成协同，文化意义才能在跨语境流动中保持可辨识的厚度。这一判断并不意味着否定技术的价值，而是强调：技术效率本身不能替代文化主体性的自觉建构。

## 句式与标点生成协议

### 句子节奏

- **长句**：用于展开概念、机制、关系、因果链或限定条件。
- **中句**：用于承接、过渡、限定或转换视角。
- **短句**：用于落定判断、标记转折或收束段落。

节奏由论证需要决定，不机械凑比例。

### 标点克制

- 少用冒号。禁止在正文中使用“问题在于：”“关键在于：”“核心观点是：”“具体而言：”等冒号小标题腔。
- 少用装饰性引号。引号只用于文献标题、直接引用或确有必要标注的概念。
- 少用破折号。破折号容易造成说明腔和AI腔。
- 句末少用感叹号。

### 段首禁忌

段落开头不能空、不能悬、不能先否定。禁用段首：

- 不是……/并非……/不能……/不应……/不要……
- 这种……/这一……/上述……/这说明……
- 关键在于……/核心是……/必须……
- 在……背景下/随着……的发展/当前/近年来（除非紧接对象关系）

段首应先给对象、关系、材料、学术位置或必要过渡。

## 每段生成后的即时自检

写完一段后，立即用以下问题自检：

1. 本段第一句是否直接呈现了对象或关系？
2. 文献是否以消化后的方式进入，而不是“有研究指出”？
3. 是否有作者主语或论文自我说明？
4. 是否有冒号小标题腔或装饰性标点？
5. 段首是否以否定、悬空指代或过早判断开头？
6. 段落是否走完了“对象 -> 关系 -> 材料 -> 机制 -> 判断”的运动？
7. 最后一句是否落到了一个具体判断，而不是口号？
8. 删除本段是否会损害全文的论证推进？

如果答案不理想，当场重写该段，不要留到后期统一润色。

## 与审计脚本的关系

`bad_draft_audit.py` 和 `scholarliness_audit.py` 保留，但只作为最后兜底。主流程是：

1. 写前生成协议；
2. 段落级大纲；
3. 逐段生成；
4. 每段生成后即时小修；
5. 最后才整体审计。

如果审计脚本发现大量问题，说明生成协议没有被遵守，应返回协议重新生成，而不是仅在文字层面修补。
