# EMARX v7.2 小节生产控制协议

本协议用于解决“整篇一次生成后像说明书、回炉又复制坏语言”的问题。完整论文不得再默认整篇生成，应按小节生产、逐节过关、失败回炉。

## 一、核心原则

1. 小节是最小生产单元。完整论文的正文按 `### （一）...` 等二级标题逐节生成。
2. 每个小节生成前必须有论证卡。论证卡只作后台控制，不能进入正文。
3. 每篇文献先绑定功能，再进入句子。引用不是证明标签，而是概念、机制、问题、材料或限制的来源。
4. 每个小节先建立问题链，从具体对象或小问题进入，再推出关系、机制、限度和大判断。
5. 每个小节生成后立即过质量门。未过门的小节不得写入全文定稿。
6. 失败稿不能全文回炉。只保留中心判断、标题链、来源映射和有效材料，回到干净小节大纲重写。

## 二、文献功能绑定

生成正文前，为每篇拟用文献建立唯一功能：

```text
文献编号：
文献题名：
唯一使用位置：
进入正文的句子功能：概念 / 机制 / 问题 / 材料 / 限制 / 反例
支撑的具体关系：
不能使用它证明的内容：
引用句草案：
```

不允许出现“先写正文、后补引用”。若一句话删去引用仍完全成立，说明该引用没有进入论证。

## 三、小节论证卡

每个小节写作前必须回答：

```text
小节标题：
本小节解释的关系：
本小节承接上一节的未尽问题：
本小节要使用的材料或文献功能：
本小节的小问题入口：
本小节的问题链：小问题 -> 关系问题 -> 机制问题 -> 限度问题 -> 大判断 -> 转出问题
本小节排除的浅层说法：
本小节必须避免的表达惯性：
本小节最后落到的判断：
下一小节为什么必须出现：
```

论证卡不要求写得长，但必须具体。若“解释的关系”只能写成“分析机遇”“提出路径”，或“小问题入口”只能写成宏大概念，说明小节尚未具备写作条件。

## 四、小节生成方式

生成小节时只提供：

- 全文中心判断；
- 当前一级标题和二级标题；
- 上一小节的最后一段；
- 当前小节论证卡；
- 当前小节需要的原文材料或文献功能；
- 下一小节的任务提示。

不要把失败稿全文或大量旧生成稿放进提示。旧稿会污染句法。

## 五、小节质量门

小节写完后，必须检查：

```text
是否有可见作者主语：
是否有综述插入套式：
是否有公式化对照：
是否有未证明强词：
是否有段首悬空指代：
是否有连续应然句：
是否有泛动词空转：
是否复述标题：
是否以宏大判断倒扣材料：
是否缺少小问题入口：
是否没有沿问题链推进：
引用是否进入句子：
段落是否只有断言：
结尾是否口号化：
```

质量门失败时，先判断失败类型：

- **语言污染**：禁用表达、AI腔、公式化句式。处理方式是重写该小节，不是替换单词。
- **论证空转**：泛动词、应然句、没有材料。处理方式是回到论证卡补关系和材料。
- **问题链断裂**：段落只是并列说明，不能从小问题推出机制和判断。处理方式是重写问题链，再按段落逐段生成。
- **以大见小**：先给宏大结论，再用材料装饰。处理方式是把具体对象、材料或局部现象前置为入口。
- **文献脱落**：引用只贴在句尾或与句子无关。处理方式是重做文献功能绑定。
- **结构无效**：小节删去后全文仍成立。处理方式是重排标题链。

## 六、失败回炉

同一小节两次出现同类硬性问题时，停止要求同一生成器继续润色，改用更小粒度：

```text
先写本小节第一段
过门
再写第二段
过门
最后合并小节
```

整篇失败时，不得全文润色。应建立新的 `section-cards.json`，逐节重写。

## 七、完成标准

一个小节可进入全文定稿，必须同时满足：

- 小节标题存在且与一级标题形成收窄关系；
- 至少两个自然段，且每段有不同论证动作；
- 有清楚的小问题入口，并能从小问题推出较大的机制判断；
- 若有引用，引用只出现一次并跟随具体使用句；
- 没有硬性禁用表达；
- 没有连续说明书式应然句；
- 泛动词有明确主语、对象、机制和条件；
- 小节最后一句回到判断，不以“具有重要意义”“形成合力”等口号收束。
- 小节删去后，全文的论证链会明显断裂。

完整论文交付前仍需运行整体 `bad_draft_audit.py`、`scholarliness_audit.py`、`citation_audit.py` 和 `audit_docx.py`。小节质量门只能减少坏稿进入全文，不能替代最终审稿。
