# EMARX Workflow

This workflow keeps the v4 research-production sequence and adds v5 length and heading-hierarchy constraints.

Use this workflow when the user asks for a full paper, research plan, deep revision, or topic development.

## 1. Local-Source-First Intake

Start from the user's workspace. Scan PDF, DOCX, Markdown, TXT, HTML, and saved web files before going online.

Build a source table:

```text
file:
type:
topic keywords:
central claim:
usable evidence:
usable concepts:
possible section:
fact risks:
relevance:
```

Use web search only when:

- local sources are absent or too thin;
- current policies, laws, data, institutional facts, or recent scholarship matter;
- local materials contain claims that need external verification;
- the user explicitly asks for online search.

## 2. Research Brief

Before drafting, produce a research brief:

```text
题目:
材料基础:
核心问题:
研究对象:
概念边界:
理论张力:
论证链条:
本地资料可支撑的判断:
资料缺口:
需核验事实:
暂不应硬写的内容:
```

Do not proceed to full drafting if the brief shows that the title cannot support a real argument. Propose a sharper thesis first.

## 3. Corpus Pattern Selection

Read `distillation-evidence.md` and select patterns instead of inventing from scratch.

At minimum choose:

- one title pattern;
- one abstract pattern;
- one introduction entry type;
- one dominant structure chain;
- two to four paragraph templates;
- one anti-false-completion rule.

Explain internally why each pattern fits the topic. Do not copy a pattern mechanically when it does not fit.

## 4. Innovation Analysis

Separate innovation into six categories:

```text
选题创新:
视角创新:
概念创新:
机制创新:
路径创新:
表达创新:
```

Then classify:

```text
真创新:
弱创新:
伪创新:
需要补材料才能成立的创新:
```

Reject false innovation such as:

- merely adding "AI", "数字化", or "新时代" to an old topic;
- renaming a known path without changing the mechanism;
- using policy vocabulary as if it were theoretical contribution;
- claiming literature gaps without checking literature.

## 4.5 Citation Planning

Before drafting any referenced paper, read `citation-protocol.md`.

Build a citation coverage table:

```text
source_id:
file:
first author:
all visible authors:
usable claim:
target sentence/section:
GB/T 7714 entry:
status:
```

If the user requires all workspace literature, every usable literature source must be mapped to exactly one sentence. If two sources share an author, report the conflict because "all sources used" and "one author only once" cannot both be satisfied without relaxing one rule.

Do not cite a source merely to satisfy coverage. Relevance remains mandatory.

## 5. Logic Framework

Use a progression chain, not a topic list.

Good chains:

```text
概念界定 -> 结构变化 -> 风险生成 -> 治理路径
理论张力 -> 机制转换 -> 主体重建 -> 实践闭环
符号呈现 -> 意义解释 -> 认同生成 -> 传播优化
材料共识 -> 理论缺口 -> 创新论断 -> 章节展开
```

Each first-level heading must answer a different theoretical question. Each second-level heading must perform a distinct analytical function.

## 5.5 Length And Heading Hierarchy

Before drafting a full paper, read `length-and-hierarchy-protocol.md`.

Build a length and hierarchy plan:

```text
target main-text characters: 10,000-12,000 unless user specifies otherwise
first-level sections: normally 4-5
second-level headings: at least two under every major body section
third-level headings: only when a section has multiple mechanisms, stages, subjects, or cases
sections at risk of being shallow:
corpus structure pattern selected:
```

Do not treat this as formatting. Heading hierarchy is part of argument depth. A full paper with only first-level headings usually becomes a list essay, even if the prose is fluent.

## 6. Drafting

Draft only after the brief and framework are stable.

Each paragraph must perform one action:

- define a concept;
- distinguish two concepts;
- reveal a tension;
- explain a mechanism;
- connect material to judgment;
- evaluate a risk;
- close a path loop.

Avoid ungrounded "importance" statements. Replace them with the exact object of value: subjectivity, legitimacy, identity, interpretive power, governance capacity, educational efficacy, or civilization dialogue.

During drafting, keep the planned length visible. Do not end the paper at 3,000-5,000 characters and call it complete. If the argument cannot sustain 10,000-12,000 characters, return to source intake and problem diagnosis instead of padding.

## 7. Review And Revision

Review the draft like an anonymous reviewer, then revise structurally.

Revision may include:

- rewriting the central thesis;
- changing section order;
- deleting shallow paragraphs;
- adding concept boundary work;
- adding mechanism explanation;
- moving unsupported claims into the fact-check list;
- changing the title and abstract after the body is sharpened.

## 8. Delivery

For full papers, produce:

- Markdown working draft when useful;
- Word `.docx` as final delivery;
- fact-check list;
- citation audit and GB/T 7714 reference list when references are used;
- short delivery note.

Do not claim Word delivery unless the file was actually created and verified.
