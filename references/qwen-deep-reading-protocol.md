# 千问逐篇精读协议

本协议用于把工作空间论文转换为可复核的逐篇证据报告。千问承担全文读取、原文定位、结构还原和细部压缩，不承担 EMARX 架构判断、总体规律定稿或最终质量结论。

## 基本原则

1. 每篇论文单独读取、单独报告，不从旧拆解报告复制结论。
2. 每篇论文均按连续页块精读并保留页块报告，再形成全篇综合。所有分块必须完成，不能抽样替代全文，也不能在综合时删除未被选中的段落证据。
3. 统一的是证据字段，不统一论文结论。不得把论文强塞进固定的五步流程、段落类型或风格标签。
4. 每项判断必须附 `[[pN:LNNNN]]` 页码行号。找不到证据就写“未确认”，不得凭题名推断。
5. 摘要、引言、正文和结论必须按原文实际结构拆解。PDF 双栏错序、刊头、作者信息、基金信息和参考文献污染必须单独标明。
6. 报告只能作为证据层。主模型必须抽检原文，才能据此形成总体规律或修改技能。
7. 千问不得手抄原文充当直接引语，只负责返回页码行号。程序依据行号从原始全文确定性回填证据，避免“近似引用”冒充原文。
8. 不得调用外部常识补充日期、作者身份或事实判断。当前时间、文献真伪和案例真实性不在本步骤判断范围内。
9. 使用 OpenAI 兼容 API 时，应显式关闭 reasoning 输出。对当前 Qwen3.5-122B-A10B 接口，脚本通过 `chat_template_kwargs: {"enable_thinking": false}` 实现；若 usage 中 `reasoning_tokens` 不为 0，应暂停全量跑并排查。
10. 标题一致性警告不是自动失败，也不是自动通过。PDF 双栏错序和报告分析标签会带来误报；但若主模型抽检发现报告把原文标题改写、补造或替换，应把该报告作为不可靠样本处理，必要时重跑。

## 每篇报告必须回答的问题

- 论文真实标题、作者、文献类型和提取噪声是什么。
- 摘要每个语义单元承担什么作用，句间如何推进，采用了哪些具体表达手法。
- 引言怎样进入问题，怎样处理政策、现象、文献和理论位置，怎样收束到正文。
- 正文有哪些真实标题层级，各部分分别推进什么，段落之间如何转折、递进、回扣或转换材料。
- 结论怎样收束，是理论抽象、边界限定、路径回扣还是简单复述。
- 论点、根据、概念、材料、反张力和结论之间形成了怎样的实际论证链。
- 长句、中句、短句在原文中分别承担什么工作；节奏如何随论证变化。
- 主语如何安排，动词如何选择，文献如何进入，过渡如何完成，判断如何落定。
- 哪些写法只属于这篇论文，哪些方法可能迁移；每项都必须有原文证据。

## 禁止输出

- “以背景打开、提出问题、分析机制、给出路径”之类可套用到任何论文的万能概括。
- “中等长度句子为主，兼顾解释与推进”之类没有例句的标签。
- “主体单元一”“主体单元二”等替代真实章节的占位名称。
- 仅给比例、词频或标签，不解释这些现象在具体论证中做了什么。
- 用同一段结论批量覆盖多篇论文。

## 质量门槛

报告至少应包含：

- 原文结构地图；
- 摘要、引言、主体、结论的分别拆解；
- 文章特有的论证链；
- 多组带页码行号的定位，及程序自动回填的逐字证据；
- 具体句法、节奏、主语、动词、过渡和引用方式分析；
- 提取噪声和不确定项。
- 逐页精读原始证据附录，覆盖原文每一页。

没有覆盖全部页码、定位行号不存在、仍使用旧模板话术、把刊头当摘要、遗漏真实二级标题或无法还原真实标题层级的报告，必须判为失败并重跑。

标题层级核查分两层：

- 机械层：脚本记录 `unmatched_claimed_source_titles`，作为抽样复核线索。
- 语义层：主模型必须回到原文判断这些 warning 是报告误写、PDF 噪声，还是脚本误报。只有确认报告误写原文结构时，才将其排除或重跑。
