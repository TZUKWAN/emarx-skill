# EMARX Fact-Check Protocol

Writing generation and fact verification are separate tasks.

## Never Invent

Never fabricate:

- policy quotations;
- leader quotations;
- law or regulation text;
- statistics;
- publication metadata;
- author names;
- journal names;
- page numbers;
- exact dates;
- institutional facts;
- real cases;
- scholar viewpoints attributed to named people;
- GB/T 7714 reference metadata.

## Local First, Web When Needed

Use local workspace materials first for:

- user's downloaded papers;
- notes;
- saved policies;
- PDFs;
- prior drafts;
- local source excerpts.

Use web verification when:

- current policies, laws, statistics, or institutional facts matter;
- a source is missing locally;
- a claim depends on recent scholarship;
- local materials may be outdated;
- the user explicitly asks for online verification.

## Fact-Risk Categories

Mark claims as:

```text
已由本地资料支持:
已联网核验:
待核验:
无法确认:
应删除或改写:
```

## Safe Writing When Unverified

If a factual claim is unverified:

- write at a conceptual level;
- avoid exact numbers and names;
- say "相关研究指出" only if actual studies are available;
- do not include fake references;
- move the item to the fact-check list.

## Final Fact-Check Table

For substantial papers, include a final table:

```text
claim:
location:
source basis:
status:
action:
```

If no verification was performed, explicitly say so.

## Reference Metadata Verification

Before writing the reference list:

- verify author names;
- verify title;
- verify document type marker such as `[J]`, `[M]`, `[D]`, `[N]`, `[EB/OL]`;
- verify journal/book/publisher/institution;
- verify year, volume, issue, and page range when available;
- do not fill missing fields by guesswork.

If metadata is missing, mark it outside the formal reference list and ask for source confirmation rather than inventing it.
