# EMARX v7.2 期刊体例与问题链协议

本协议用于修复四类实测问题：引言默认过长、摘要偏理科化、一级章下缺少小序、正文论证分散且收束不够。它要求在写作前先确定期刊体例画像，再用问题链组织正文。

## 一、期刊体例画像

完整论文写作前，先根据用户指定期刊、目标刊物风格或锚定论文建立体例画像。没有明确期刊时，从工作空间锚定论文中抽取相近体例，不得默认所有论文都使用长引言。

画像至少包含：

```text
目标期刊或参照论文：
引言类型：短引言 / 标准引言 / 政策阐释-文献对话-问题引出
摘要类型：人文社科报道式 / 理论思辨式 / 政策问题式
一级章下是否有小序：
正文常见段落长度：
标题层级习惯：
结论长度和收束方式：
```

体例画像是后台工作，不得写进正文。

## 二、引言模式

引言的任务是把读者带到研究问题。长度服从期刊体例，不以“越充分越好”为标准。

### 短引言

适用于引言通常只有一段的期刊或评论型理论文章。一般 300-600 个中文字符。运动方式：

```text
对象变化 -> 问题张力 -> 学术切口 -> 中心判断
```

短引言不写完整文献综述，不铺陈宏大背景，不介绍全文结构。

### 标准引言

适用于多数中文人文社科论文。一般 700-1200 个中文字符。运动方式：

```text
现实或理论语境 -> 既有讨论的有效解释 -> 尚未打通的关系 -> 本文问题意识
```

文献只能被消化为问题来源、概念来源或解释限度，不写成“甲认为、乙认为”的综述清单。

### 政策阐释—文献对话—问题引出

适用于政策语境较强、编辑期待现实关切的期刊。一般 1000-1800 个中文字符。运动方式：

```text
政策命题或时代任务 -> 学界已有解释 -> 解释缺口或现实新变化 -> 需要回答的问题
```

政策阐释要问题化。政策文本不能只作为正确性背书，必须转化为论文要解释的对象关系。

## 三、摘要写法

中文人文社科摘要不是理科 IMRaD 摘要。不得写成：

```text
目的 -> 方法 -> 结果 -> 结论
```

也不得写成：

```text
本文研究什么 -> 本文从几个方面分析 -> 本文提出对策
```

更合适的摘要运动是：

```text
对象处境 -> 问题张力 -> 解释关系 -> 判断落点 -> 必要的路径指向
```

摘要应当像一段压缩后的学理论证，读者读完后能知道对象为什么成为问题、分析抓住了什么关系、判断落在何处。摘要不列目录，不自我介绍，不使用“目的、方法、结果、结论”等理科标签。

## 四、章下小序

正文一级章节下、第一个二级标题前，通常应有一段小序。小序不是章节摘要，也不是“本章将从三个方面展开”的说明书。

小序承担三项功能：

```text
承接上一层论证或引言留下的问题；
说明本章要打开的内部关系；
引出本章第一个二级小节。
```

小序一般 120-250 个中文字符。它要自然进入，不要写成清单。

合格小序示例：

```text
智能生成技术首先改变了文化意义被组织和转译的方式。只有看清这种入口变化，才能解释后续的受众反馈、符号生成和主体性风险。因此，分析应从技术进入传播行动结构的方式开始。
```

## 五、问题链写法

正文小节不要先抛大概念再找材料证明。优先用“小问题”进入，从具体对象、材料、现象或局部关系出发，逐步推到较大的理论判断。

每个二级小节写作前，先建立问题链：

```text
小问题：本节从哪个具体对象或现象进入？
关系问题：这个小对象暴露了哪组关系？
机制问题：这组关系为什么会发生？
限度问题：它在哪些条件下失效、偏移或产生风险？
大判断：由此能够推出什么更大的学理判断？
转出问题：下一节为什么必须继续？
```

正文不直接写出这些问题。问题链只用于控制段落推进。

## 六、以小见大

“以小见大”不是把案例写小，而是让具体对象承担理论推进功能。一个小点能够进入正文，必须能回答一个更大的关系问题。

优先路径：

```text
具体材料或局部现象 -> 关系变化 -> 机制解释 -> 理论判断
```

谨慎使用：

```text
宏大判断 -> 一般解释 -> 举例说明
```

后一种写法容易变成以大见小，编辑会看到先验结论和材料装饰。除非题目本身要求宏观理论推演，否则正文段落应从具体对象、具体关系或具体材料起笔。

## 七、三段论的正确用法

三段论不是三段清单。它是小点引出大点的推理链：

```text
第一段：从具体对象进入，说明小问题为什么成立。
第二段：解释小问题背后的机制，处理文献或材料。
第三段：把机制推向更大的判断，并为下一节留下必要问题。
```

每个小节可以是三段，也可以是四段或五段。段数服从问题链，不按固定模板凑数。

## 八、收束规则

每个自然段、每个二级小节、每个一级章节都要有收束。收束不是喊口号，也不是重复标题。

合格收束应当说明：

```text
本段或本节推进了什么判断；
这个判断改变了前文哪一层理解；
它为什么要求下一段或下一节继续。
```

若一节写完后只能总结为“很重要、要加强、要提升”，说明论证没有聚焦，需要回到问题链重写。
