# EMARX 理论增量判断协议

## 一、为什么需要理论增量判断

马理论顶刊审稿的核心问题之一是：这篇论文比已有研究多解释了什么？如果论文只是把既有观点重新排列组合，或者只是换一种说法描述常识，就难以通过外审。理论增量判断强制在动笔前和定稿后明确回答这个问题。

## 二、理论增量的五种形式

### 1. 解释新现象

已有理论框架无法充分解释的新现象、新经验或新文本，论文提供了新的解释路径。

> 例：数字平台劳动不是传统雇佣劳动，也不是完全独立的个体劳动，论文提出"算法中介下的不完全劳动关系"来解释其特殊性。

### 2. 修正既有解释

已有研究能够解释某类问题，但在新条件下失效，论文指出其失效点并提出修正。

> 例：传播学既有研究能够解释内容如何抵达受众，但难以解释模型在生成环节如何改变文化意义的组织方式，论文引入"意义解释权"概念来修正。

### 3. 澄清概念边界

某个核心概念在既有研究中被混用或滥用，论文通过概念辨析恢复其分析效力。

> 例："人民""群众""民众"在相关研究中被混用，论文通过历史语义分析澄清三者的政治内涵和适用语境。

### 4. 揭示新机制

论文发现连接现象与判断之间的中间机制，使因果关系或作用方式变得更清楚。

> 例：论文揭示"红色记忆空间"通过物质陈列、仪式参与和叙事引导三重机制培育政治认同。

### 5. 建立新关系

论文在两个看似无关的研究对象或理论传统之间建立联系，产生新的问题域。

> 例：论文把马克思的"时间经济"思想与当代平台经济中的"注意力经济"联系起来，揭示数字资本主义的时间规训新形式。

## 三、理论增量判断模板

动笔前必须填写：

```text
研究对象：__________
已有研究 A 能够解释：__________
已有研究 A 无法解释的是：__________
当对象出现 __________ 变化时，A 的 __________ 前提不再成立；
本文引入/回到/重新解释 __________，以说明 __________；
本文的理论增量属于：□ 解释新现象 □ 修正既有解释 □ 澄清概念边界 □ 揭示新机制 □ 建立新关系
本文的核心判断是：__________
如果删除本文的核心判断，已有研究是否仍能完整解释该对象？□ 是 □ 否
```

最后一问若回答"是"，说明理论增量不足，应重新选题或重新聚焦问题。

## 四、问题尖锐度评估

好问题通常具有以下特征：

1. **反直觉**：与通行看法或政策话语表面意思形成张力；
2. **有具体靶子**：能够指出某个具体研究、某个具体观点或某个具体论证的不足；
3. **可证伪**：论文判断可以被材料证实或反驳，而不是永远正确；
4. **非做不可**：如果这个问题不解决，后续研究或实践会受到影响；
5. **能用三句话说服编辑**：在简短交流中能让非本领域专家理解其重要性。

## 五、常见"伪增量"陷阱

### 1. 视角替换

> "已有研究多从 A 角度切入，本文从 B 角度切入。"

问题：视角替换不等于理论推进。必须说明 B 角度为什么能解释 A 角度解释不了的问题。

### 2. 材料更新

> "已有研究主要基于传统案例，本文使用最新案例。"

问题：新材料只有揭示新机制或新关系时才有增量。

### 3. 对象平移

> "已有研究讨论的是城市，本文讨论农村。"

问题：平移对象必须说明城市理论在农村语境下为何失效。

### 4. 概念重命名

> "已有研究使用'认同'，本文使用'共同体感'。"

问题：新术语必须有新内涵，否则只是同义反复。

### 5. 政策附和

> "本文响应中央关于……的号召。"

问题：政治响应需要转化为具体理论问题，否则只是政策宣传。

## 六、在小节论证卡中落实理论增量

每个小节论证卡必须回答：

- 本小节要推进的理论问题是什么？
- 本节之前，已有解释在哪个环节不够？
- 本小节将引入什么新材料、新概念或新机制？
- 本小节完成后，读者对问题的理解比读之前增加了什么？
- 如果删除本小节，全文的理论推进是否会断裂？

## 七、审稿阶段的增量复核

初稿完成后，用以下问题复核：

- [ ] 论文是否明确指出了至少一个需要回应的具体学术观点？
- [ ] 论文是否说明了该观点能解释什么、不能解释什么？
- [ ] 论文的核心判断是否能用一句话概括，且包含具体关系或机制？
- [ ] 论文是否避免了"视角替换""材料更新""对象平移""概念重命名"等伪增量？
- [ ] 论文结论是否提出了可被进一步讨论、反驳或使用的理论判断？

## 八、与 EMARX 其他协议的关系

- 理论增量判断以 `theoretical-problem-checklist.md` 为基础；
- 理论增量的实现依赖 `scholarliness-protocol.md` 中的概念纪律和论证推进；
- 理论增量的表达需要 `style-protocol.md` 的平实语体；
- 理论增量必须有 `citation-fact-protocol-v7.md` 的事实和文献支撑。

## 九、失败信号

出现以下情况说明理论增量不足：

- 核心判断删除后，论文仍像一篇完整的综述；
- 无法在三句话内向同行说明本文的新意；
- 各节互换顺序后仍然成立；
- 结论只剩下"具有重要意义"，没有具体理论判断；
- 审稿人可能问"这篇文章到底想说什么"而难以回答。
