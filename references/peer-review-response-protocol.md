# EMARX 审稿回应协议

## 一、适用范围

本协议用于论文投出后收到外审意见时的修改与回应。顶刊论文往往需要经过多轮外审和修改，能否有效回应审稿意见直接影响录用结果。

## 二、基本原则

1. **尊重但不盲从**：认真对待每一条意见，但不必接受所有意见。对不合理的意见要有礼貌地解释。
2. **修改优先于辩解**：能用修改正文解决的问题，不要只在回应信中辩解。
3. **逐条回应**：对每条审稿意见都要给出明确回应，不能遗漏。
4. **证据说话**：修改和回应都要有文本依据、文献依据或事实依据。
5. **保持学术礼貌**：即使不同意审稿人意见，也要避免情绪化表达。

## 三、审稿意见分类

收到审稿意见后，首先进行分类：

| 类型 | 特征 | 处理方式 |
|---|---|---|
| 根本性问题 | 质疑论文的核心判断、理论框架或问题意识 | 必须认真考虑，可能需要大幅修改甚至重写 |
| 论证缺陷 | 指出某段论证逻辑不严密、材料不足 | 补充材料、修正推理或调整结论 |
| 文献遗漏 | 建议补充重要文献 | 补充阅读并在正文中合理引用 |
| 事实错误 | 指出数据、年份、引文等错误 | 核实后修正 |
| 表述问题 | 指出语言、格式、标点问题 | 直接修改 |
| 误解性意见 | 审稿人误解了论文意图 | 在回应中解释，并考虑是否在正文中说得更清楚 |
| 不合理意见 | 与论文定位不符或超出范围 | 礼貌说明不能接受的原因 |

## 四、修改步骤

### 第一步：通读所有意见

- 不要急于逐条回应；
- 先判断审稿人的总体评价和主要关切；
- 标记出必须修改的根本性问题。

### 第二步：制定修改计划

- 列出所有需要修改的位置；
- 判断哪些修改会相互影响；
- 优先处理根本性问题，再处理局部问题。

### 第三步：修改正文

- 使用修订模式或高亮标注修改处；
- 保持论文整体风格一致；
- 新增内容要与原有论证有机融合，不能生硬拼接。

### 第四步：撰写回应信

- 回应信应包括：感谢、总体修改说明、逐条回应；
- 每条回应应包括：审稿意见原文或摘要、修改说明、修改位置。

## 五、回应信模板

```text
尊敬的编辑、审稿专家：

非常感谢贵刊和审稿专家对本文提出的宝贵意见。我们认真阅读并逐条思考了所有意见，对论文进行了相应修改。现将主要修改说明如下：

一、总体修改说明

1. 针对审稿专家提出的……问题，我们重新梳理了论文的理论框架，调整了……
2. 补充了……方面的文献和材料，强化了……论证。
3. 对全文文字进行了进一步打磨，修正了……等表述。

二、逐条回应

意见 1："……"

回应：感谢审稿专家的意见。我们认为……。已在论文第 X 页第 X 段修改为……。

意见 2："……"

回应：审稿专家指出的……问题确实存在。我们已补充……材料，并调整了……判断。具体见论文第 X 页第 X 段。

意见 N："……"

回应：对于这一意见，我们认为……。理由是……。因此未做修改，但已在第 X 页增加了说明，以避免误解。

再次感谢审稿专家的辛勤工作！

此致
敬礼！

作者
日期
```

## 六、常见审稿意见及回应策略

### 1. "问题意识不够明确"

**修改策略**：
- 在引言中更清晰地提出理论问题；
- 压缩背景介绍，突出矛盾；
- 明确指出现有解释的失效点。

### 2. "理论贡献不足"

**修改策略**：
- 补充与具体学术观点的对话；
- 明确说明本文比已有研究多解释了什么；
- 调整结论，提出更具体的理论判断。

### 3. "材料支撑不足"

**修改策略**：
- 补充案例、数据或文献；
- 说明材料来源和可靠性；
- 降低过度概括的判断强度。

### 4. "概念使用不清晰"

**修改策略**：
- 在关键位置增加概念界定；
- 检查全文概念使用是否一致；
- 区分相近概念。

### 5. "结构需要调整"

**修改策略**：
- 重新设计一级标题的逻辑关系；
- 确保前一部分产生后一部分的论证必要性；
- 删除或合并重复章节。

### 6. "语言需要打磨"

**修改策略**：
- 逐段检查 AI 腔、口号化、套话；
- 调整句长节奏；
- 减少抽象名词堆砌。

### 7. "建议补充某文献"

**修改策略**：
- 尽量找到并阅读该文献；
- 如果确实相关，在正文中合理引用；
- 如果确实不相关或无法获取，礼貌说明原因。

## 七、不能接受的意见如何回应

对于确实不能接受的意见，回应时应：

1. 先感谢审稿人提出该意见；
2. 说明自己的考虑和理由；
3. 提供文献或逻辑依据；
4. 如果可能，在正文中增加说明以避免其他读者产生同样误解。

示例：

> 感谢审稿专家的建议。我们考虑过将本文扩展为对……的全面研究，但这样会偏离本文聚焦……的核心问题。本文的目标是澄清……机制，而非对……进行全景式考察。为避免误解，我们已在引言中进一步限定了研究范围。

## 八、修改后的复核

提交修改稿前必须复核：

- [ ] 是否逐条回应了所有审稿意见；
- [ ] 正文中是否真正落实了承诺的修改；
- [ ] 新增内容是否与全文风格一致；
- [ ] 修改是否引入了新的政治、事实或逻辑问题；
- [ ] 回应信是否礼貌、清晰、有依据；
- [ ] 是否按要求提供了修改稿、回应信和原稿对照。

## 九、失败信号

出现以下情况说明回应失败：

- 只在回应信中辩解，不修改正文；
- 对关键意见敷衍了事；
- 修改后论文反而更混乱；
- 态度生硬，与审稿人对抗；
- 遗漏重要审稿意见。

## 十、与 EMARX 其他协议的关系

- 审稿回应以 `review-rubric.md` 的审稿标准为前提；
- 修改后的论文必须重新通过 `section-production-gate-v71.md` 和 `scholarliness-protocol.md`；
- 涉及政治、事实、引用的修改必须回到 `political-compliance-protocol.md` 和 `citation-fact-protocol-v7.md` 重新核验。
