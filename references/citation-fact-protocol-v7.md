# EMARX v7 引用与事实核查协议

本协议处理两个问题：哪些话可以写，哪些话必须有来源；文献如何进入论文，才不变成综述插入和参考文献堆砌。

## 一、基本原则

1. 写作生成和事实证明必须分开。模型可以提出判断，但判断中涉及事实、政策、数据、年份、机构、人物、案例、直接引语和文献观点时，必须回到来源核验。
2. 千问、脚本、旧报告和模型摘要都不是事实来源。它们只能帮助定位原文，不能替代原文、政策文件、正式出版信息或权威网页。
3. 文献不是装饰性标签。进入正文前，先明确它在论证中的功能：提供概念、界定问题、说明机制、提供材料、构成反例、限定条件或形成对话对象。
4. 引用必须贴住被它支撑的那句话。哪句话使用了某个文献，引用编号就放在该句后面，不集中堆到段落末尾。
5. 一篇文献只引用一次，一个作者只出现一次。若用户要求使用工作空间全部文献，而文献数量、同作者重复或相关性不足导致冲突，必须如实说明并给出取舍方案，不能用虚假引用制造合规。

## 二、事实分级

写作前建立事实清单，按风险分为三类。

高风险事实必须逐项核验并记录来源：

- 政策名称、发布时间、发布机构和原文表述；
- 数据、排名、比例、用户量、年份和时间顺序；
- 机构、平台、人物、会议、项目和案例细节；
- 直接引语、文献题名、作者、期刊、卷期、页码；
- 具有时效性的现实判断。

中风险事实应优先核验，无法核验时降低表述强度：

- 对某类研究趋势的概括；
- 对技术功能、平台机制和传播效果的描述；
- 对历史过程、制度安排和社会现象的概括性判断。

低风险内容可以由论证推出，但不能伪装成事实：

- 概念辨析；
- 理论推演；
- 对材料意义的解释；
- 由前文论证得出的判断。

## 三、来源优先级

本地工作空间优先。先读用户提供的论文、政策、草稿、资料包和案例材料；本地没有、材料过时或必须核验最新事实时，再使用网络。

来源优先级依次为：

1. 政策、法律、规划、白皮书等官方原文；
2. 正式发表论文、专著、期刊页面、数据库元数据；
3. 权威机构、平台官方公告和公开报告；
4. 用户工作空间中的可核验材料；
5. 新闻报道和二手材料。

若只能找到二手来源，正文应降低确定性，不写成原始事实。

## 四、来源到论点映射

完整论文起草前建立 `source-claim-map`，每条记录至少包含：

- 来源文件或网页；
- 来源哈希、页码、段落或 URL；
- 文献作者和年份；
- 只允许使用的一处正文位置；
- 该来源支撑的具体句子；
- 支撑类型：概念、事实、机制、案例、反例、限制、对话；
- 是否与其他文献同作者冲突；
- 元数据是否足够生成 GB/T 7714 参考文献。

正文写作时只使用已经映射的位置。若写作过程中新增事实主张，应先补映射和核验，再进入正文。

## 五、正文引用方式

禁止用下列方式插入文献：

- “有研究指出……”
- “已有研究认为……”
- “相关研究表明……”
- “学者认为……”
- “国内外研究普遍认为……”

这些句式通常把论文写成综述。正确做法是让文献成为本文论证的一部分。

可接受的进入方式：

- 概念来源：把来源的概念转化为本文分析对象的边界，并在使用句后标注引用。
- 机制来源：用来源说明某种关系如何成立，再把它接入本文对象。
- 材料来源：用来源提供具体事实、案例或文本细节，并紧随事实句标注引用。
- 限制来源：说明既有解释能说明什么，也说明其无法解释本题的哪一环。
- 对话来源：将来源作为论证推进的台阶，而不是列举名单。

引用编号按正文首次出现顺序排列。正文中不得出现 `[1][2][3]` 这类堆叠引用，不得用一个句子背负多篇文献来绕开“一文献一次”。

## 六、参考文献

文末参考文献按正文引用顺序排列，不按作者姓氏排序。每条参考文献必须能回到真实来源。

GB/T 7714 基本形态：

- 期刊：作者. 题名[J]. 刊名, 年, 卷(期): 页码.
- 专著：作者. 书名[M]. 出版地: 出版社, 年.
- 学位论文：作者. 题名[D]. 保存地: 保存单位, 年.
- 报纸：作者. 题名[N]. 报纸名, 出版日期(版次).
- 电子资源：作者或机构. 题名[EB/OL]. 发布或更新日期[引用日期]. URL.

元数据不完整时不得编造页码、卷期、出版地或日期。可以先标注“元数据待核”，并在交付说明中列为残余风险；正式交付前应补齐或删除该来源。

## 七、核查流程

1. 写作前：建立来源清单和来源到论点映射。
2. 逐段写作时：每出现一个事实句或文献句，确认它已映射到唯一来源。
3. 初稿后：运行 `scripts/citation_audit.py` 检查编号、重复作者、重复来源、堆叠引用、段尾堆砌、综述插入句和基本 GB/T 7714 形态。
4. 事实核查：逐项检查高风险事实，记录可核验来源和无法核验项。
5. 主模型语义复核：抽查引用是否真正支撑其前一句，不能只依赖脚本。
6. DOCX 生成后：确认正文引用为上标，参考文献编号不是上标，参考文献采用悬挂缩进。

脚本通过只说明机械格式暂时合格，不说明文献真的支撑论点。语义支撑必须由主模型回到来源复核。

## 八、失败信号

出现以下情况应退回重写，而不是简单替换词语：

- 正文像文献综述，连续介绍多个作者观点；
- 引用只出现在段落末尾，与具体句子脱节；
- 为了“用完所有文献”把无关文献硬塞进正文；
- 同一作者多篇文献被重复引用却没有说明冲突；
- 参考文献中出现无法验证的作者、刊名、年份或页码；
- 政策、数据和案例没有来源，却被写成确定事实；
- 文献功能与论证位置不匹配，例如用案例来源支撑理论判断。

引用的目标不是数量漂亮，而是让每个判断都有来路、每个来源都有用处、每个编号都能被追问。
