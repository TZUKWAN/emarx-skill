# EMARX 概念台账协议

## 一、什么是概念台账

概念台账是论文核心概念的内部管理表。它要求写作者在开题和写作过程中明确：每个核心概念在本文中的精确定义是什么、来自哪里、与相近概念如何区分、在论证中承担什么功能。

概念台账不进入正文，但它能防止概念漂移、概念堆砌和概念误用——这是马理论论文学理性的大敌。

## 二、哪些概念需要进台账

至少包括以下三类：

1. **题目中的核心概念**：如"人民主体性""文化主体性""智能传播""文化记忆"等。
2. **跨学科借用概念**：如把传播学的"算法"、认知科学的"具身"、信息科学的"模型"引入马理论分析时。
3. **自定义或重新定义的概念**：如果论文对某个常见概念给出了特殊用法，必须记录。

## 三、台账格式

每个核心概念记录以下内容：

```text
概念：人民主体性
本文定义：在中华文化国际传播中，人民主体性指人民群众作为文化意义的生产者、解释者和受益者的统一地位。
概念来源：马克思主义群众观 + 习近平文化思想中的主体性论述
相近概念区分：
  - 与“人民性”的区别：人民性侧重价值立场，人民主体性侧重主体地位和能力。
  - 与“文化主体性”的区别：文化主体性侧重文化本身的主体地位，人民主体性侧重人民作为文化主体的实践位置。
论证功能：用于说明智能传播时代文化传播不能只靠技术效率，而要回到人民的文化解释权和参与权。
使用边界：本文主要在“文化生产—传播—接受”的全链条中使用，不用于政治哲学中的主权讨论。
首次出现位置：引言第二段
相关概念：文化主体性、智能传播、解释权
```

## 四、概念稳定性的三项纪律

### 1. 全文同一性

同一概念在全文中内涵外延必须一致。不能前半部分把"主体性"理解为"文化自身的主体地位"，后半部分又理解为"传播者的能动性"。

### 2. 跨学科概念转译

从其他学科借来的概念，必须说明它在本文马理论语境中的转译方式。不能直接把传播学、计算机科学的概念当成不证自明的工具。

- 弱：算法改变了文化传播。
- 强：算法在这里指平台根据用户反馈对内容进行排序和推荐的机制，它改变了文化内容从生产者到接受者的流通方式。

### 3. 自定义概念克制

不要为了让论文显得高深而造新词。如果确实需要新术语，必须满足：
- 现有概念无法准确表达；
- 新术语有清晰定义；
- 新术语在论证中反复使用并承担实际功能。

## 五、概念台账与正文写作

概念台账在以下时刻被调用：

1. **引言写作前**：确认题目核心概念的定义和边界。
2. **每一小节写作前**：确认本节使用的关键概念是否与台账一致。
3. **审稿时**：抽查核心概念是否在全文保持同一性。
4. **遇到概念争议时**：回到台账，明确本文用法，不被其他用法带跑。

## 六、常见错误

- **概念漂移**：同一概念前后用法不同，读者无法判断其精确含义。
- **概念堆砌**：把几个大概念堆在一起，但没有说明它们之间的关系。
  - 弱：主体性、解释关系、治理秩序和价值协同构成了智能传播的结构基础。
  - 强：人民主体性为智能传播提供了价值立足点，算法治理秩序为主体性的实现提供制度条件，二者通过解释权的分配形成张力。
- **概念神秘化**：用抽象名词替代具体关系。
  - 弱：技术赋能了文化主体性的生成机制。
  - 强：智能生成工具使普通用户能够参与文化内容的再创作，这扩大了文化生产的主体范围，但也使专业传播者的解释权威受到挑战。
- **跨学科概念未转译**：直接使用其他学科术语而不说明在马理论语境中的含义。

## 七、检查清单

写作和审稿时确认：

- [ ] 题目中的每个核心概念都有明确定义；
- [ ] 跨学科概念已经过转译；
- [ ] 同一概念在全文中内涵外延一致；
- [ ] 没有为装饰而造的自定义概念；
- [ ] 概念之间的关系已经说清楚，不是简单堆砌。
