# EMARX 逻辑推理链协议

## 一、总则

论文的逻辑推理链是读者能否跟随作者从对象进入判断的关键。它不是形式上的标题排列，而是问题、材料、机制、判断之间的真实运动。本协议规定标题之间、段落之间、句子之间应如何建立和保持逻辑关联。

逻辑推理链有三个层次：

1. **宏观层**：一级标题之间的递进关系；
2. **中观层**：二级标题和段落之间的承接关系；
3. **微观层**：句子之间的推进关系。

每个层次都必须让读者感到：上一环节如果不成立，下一环节就无法展开。

## 二、一级标题之间的逻辑关联

### 核心原则

一级标题之间必须构成**递进关系**，而不是并列材料筐。递进意味着：前一节的结论是后一节展开的前提，或者前一节未解决的问题正是后一节要处理的问题。

### 检验方法

**互换测试**：任意两节互换顺序后，如果全文仍然成立，说明结构只是并列罗列，需要重新设计。

**删除测试**：删除任一节，如果全文论证没有明显断裂，说明该节是可有可无的装饰。

### 常见递进模式

```text
对象变化 -> 机制展开 -> 风险显影 -> 路径回应 -> 理论回归
概念关系 -> 理论张力 -> 机制推导 -> 边界校正 -> 抽象提升
问题生成 -> 深层原因 -> 具体表现 -> 条件转换 -> 价值收束
历史形成 -> 叙事机制 -> 认同功能 -> 当代困境 -> 实践转化
```

选择哪种模式，由题目、材料和锚定论文共同决定。

### 一级标题之间的过渡

每个一级标题下应有一段**章下小序**，完成三个动作：

```text
承接上一层问题或上一章结尾；
说明本章要打开的内部关系；
引出本章第一个二级标题。
```

章下小序不是章节摘要，也不是"本章将从以下几个方面展开"的说明书。它要让读者感到：因为上一章推进到了某一点，所以必须进入这一章。

## 三、二级标题之间的逻辑关联

### 核心原则

二级标题是把一级标题内部的论证动作拆开。它们之间应当形成**从小问题到大判断的推进**，或者从一种维度到另一种维度的递进。

### 二级标题之间的关系类型

| 关系类型 | 说明 | 示例 |
|---|---|---|
| **递进** | 从前提推到结论 | 概念界定 -> 机制展开 -> 理论判断 |
| **分层** | 从宏观到微观，或从抽象到具体 | 宏观结构 -> 中观机制 -> 微观实践 |
| **分主体** | 不同主体承担不同功能 | 国家层面 -> 平台层面 -> 个体层面 |
| **分阶段** | 历史或过程的先后阶段 | 形成期 -> 发展期 -> 转型期 |
| **对比** | 揭示张力或矛盾 | 效率逻辑 -> 意义逻辑 |

### 二级标题下的小问题链

每个二级标题下必须建立问题链：

```text
小问题：本节从哪个具体对象或现象进入？
关系问题：这个小对象暴露了哪组关系？
机制问题：这组关系为什么会发生？
限度问题：它在哪些条件下失效、偏移或产生风险？
大判断：由此能够推出什么更大的学理判断？
转出问题：下一节为什么必须继续？
```

正文不直接写出这些问题。问题链只用于控制段落推进。

## 四、段落之间的逻辑关联

### 段落衔接的四种方式

1. **问题推进**：上一段提出一个问题，本段回答或深化；
2. **机制展开**：上一段描述现象，本段解释机制；
3. **范围转换**：上一段讨论一般情况，本段讨论特殊情况或反例；
4. **判断提升**：上一段给出具体判断，本段提升到更抽象层次。

### 段落首句的功能

段首不应悬空。每个段落开头应完成以下功能之一：

- 承接上一段的判断，并指出未尽之处；
- 引入新的具体对象或材料；
- 澄清概念边界；
- 点名学术争论中的某一位置。

### 段落之间的过渡词

允许使用的过渡方式：

- "这意味着……"（用于从现象到机制）
- "问题在于……"（用于从描述到问题）
- "由此观之……"（用于从材料到判断）
- "但……""然而……"（用于引入张力或反例）
- "因此……"（用于从机制到判断，必须有真实因果支撑）

禁止用过渡词制造虚假逻辑：

- "综上所述"（正文内部，用于提前收束）
- "由此可见"（如果前文并没有推出该判断）
- "因此"（没有真实因果关系的强转折）

## 五、句子之间的逻辑关联

### 句子推进的基本模式

每个句子应比前一个句子多推进一点。常见推进方式：

```text
对象 -> 关系 -> 机制 -> 判断
事实 -> 解释 -> 推论 -> 限定
```

### 避免逻辑跳跃

以下情况属于逻辑跳跃：

- 从现象直接跳到"具有重要意义"；
- 从 A 学者观点直接跳到 B 学者观点，没有说明二者关系；
- 从材料描述直接跳到政策建议；
- 从"存在问题"直接跳到"应该加强"，缺少机制分析。

### 让动词承担推进

句子之间的逻辑关系应通过动词和判断词呈现，而不是靠连接词包装。例如：

> 弱：数字平台提高了效率，因此也带来了风险。
> 强：数字平台通过压缩配送时间提高了履约效率，但时间约束的强化同时把劳动节奏的控制权从劳动者转移给算法系统。

## 六、以小见大：材料到判断的推理链

"以小见大"不是把案例写小，而是让具体对象承担理论推进功能。优先路径：

```text
具体材料或局部现象 -> 关系变化 -> 机制解释 -> 理论判断
```

谨慎使用：

```text
宏大判断 -> 一般解释 -> 举例说明
```

后一种写法容易变成先验结论加材料装饰。

## 七、收束与过渡

### 段落收束

每个自然段结尾应落到具体判断，并为下一段留下过渡。段落收束要回答：

- 本段推进了什么判断？
- 这个判断改变了前文哪一层理解？
- 它为什么要求下一段继续？

### 小节收束

每个二级小节最后一段必须完成收束：

- 本节推进了什么判断；
- 改变了前文哪一层理解；
- 为什么下一节仍有必要。

### 章节收束

每个一级章节末尾应自然引出下一章的问题。可以通过章下小序在下一章开头完成承接。

## 八、路径部分的逻辑关联

路径部分不是行动清单，而是论证的延伸。它必须从前文已经证明的机制、风险和条件中推出。

合格路径的逻辑结构：

```text
已诊断的机制 -> 需要改变的条件 -> 行动者及其行动 -> 该行动如何回应机制 -> 理论收束
```

每个路径都要对应一个前文已经证明的机制或风险。不能先写"应当加强语料建设"，再倒补理由。

## 九、常见逻辑断裂及修复

| 断裂类型 | 表现 | 修复方法 |
|---|---|---|
| 标题并列 | 一级标题互换后仍然成立 | 重新设计，使前节成为后节的前提 |
| 材料脱节 | 案例描述与理论判断之间没有关系 | 在案例后加入机制分析，说明案例能解释什么 |
| 跳跃收束 | 段落以"具有重要意义"结束 | 把意义具体化，说明对哪个机制或关系的理解有何推进 |
| 路径悬空 | 对策与正文机制无关 | 删除或重写路径，使其回应前文已证明的机制 |
| 综述堆砌 | "甲认为、乙认为"后直接进入本文观点 | 把文献消化为问题来源、概念来源或限制来源 |
| 概念漂移 | 同一概念在不同段落用法不同 | 回到概念台账，统一概念边界 |

## 十、逻辑推理链自检清单

大纲和初稿完成后，逐项检查：

### 宏观层

- [ ] 一级标题之间是否形成递进关系？
- [ ] 任意两节互换顺序后，全文是否仍然成立？
- [ ] 删除任一节，全文论证是否明显断裂？
- [ ] 每个一级标题是否通过章下小序承接上一章问题？

### 中观层

- [ ] 二级标题是否真正收窄一级标题？
- [ ] 二级标题之间是否形成推进关系？
- [ ] 每个二级标题下是否建立了小问题链？
- [ ] 每个二级小节是否从小问题推进到大判断？
- [ ] 小节结尾是否为下一节留下必要问题？

### 微观层

- [ ] 每个段落首句是否直接呈现对象或承接上文？
- [ ] 段落之间是否存在逻辑跳跃？
- [ ] 句子之间是否通过动词和判断推进？
- [ ] 段落结尾是否落到具体判断，而不是口号？
- [ ] 路径部分是否回应了前文已证明的机制？
