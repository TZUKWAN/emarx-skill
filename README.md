# EMARX

EMARX 是一个面向 Codex 的中文学理思辨论文写作与 Word 交付技能。v3 版本的重点不是“生成一篇看起来像论文的文章”，而是强制执行写前诊断、核心论断、概念辨析、机制分析、学术语体控制和 `.docx` 交付。

它尤其适合马克思主义理论、思想政治教育、文化记忆、中华文化国际传播、主体性、人民性、中国式现代化、人工智能哲学、技术批判以及相邻人文社科议题。

## v3 解决的问题

v3 针对一次实际写作实验中暴露的问题进行了重构：

- 默认只输出聊天文本，没有生成 Word 文档
- 论证停留在泛泛而谈，容易人云亦云
- 语言像普通评论或申论，缺少学术质感
- 缺少反常识判断、理论张力和深度洞察

因此，v3 将“深度”和“交付”写成硬规则，而不是可选建议。

## 核心能力

EMARX 可以辅助完成：

- 论文选题与问题意识生成
- 标题拟定与标题优化
- 摘要写作
- 引言设计
- 三级提纲与章节结构搭建
- 正文段落写作与重写
- 概念辨析与理论阐释
- 技术、文化、主体性、传播机制等议题的深度分析
- 学术语体润色
- 浅层表达打回与重写
- Word `.docx` 论文交付

## v3 写作协议

当用户要求“写论文”“写一篇论文”“成文”“稿件”或类似完整论文任务时，EMARX 默认执行以下流程：

1. 建立写前诊断卡：核心论断、核心概念、理论张力、机制链条、反浅层风险、需核验事实。
2. 先生成一个非平庸中心判断，优先采用“不是 A，而是 B”的论证结构。
3. 明确核心概念的含义、边界和全文功能，避免概念混用。
4. 选择一个主导结构，而不是把“机遇、挑战、路径”机械铺开。
5. 正文每一段必须执行一个理论动作：界定、辨析、揭示张力、机制推演、价值判断或路径闭环。
6. 对“提高效率”“扩大影响”“加强建设”“优化路径”等泛词进行机制化改写。
7. 对未核验的引文、数据、政策原文和真实案例保持克制，不虚构来源。
8. 完整论文默认生成 Word `.docx` 文件，并在聊天中提供简要预览和验证限制。

## 安装方式

使用 Codex skill installer 从本仓库安装：

```bash
python ~/.codex/skills/.system/skill-installer/scripts/install-skill-from-github.py --repo TZUKWAN/emarx-skill --path . --name emarx
```

安装后重启 Codex，使技能被重新发现。

## 使用边界

EMARX 可以生成学术写作结构、理论论证路径和成熟表达范式，但不负责自动验证现实事实。

不要用 EMARX 虚构：

- 引文
- 作者
- 期刊信息
- 页码
- 政策原文
- 统计数据
- 需要来源支撑的事实判断

当论文涉及最新政策、近期研究、法律法规、机构事实、真实引文或统计数据时，必须另行使用可靠来源核验后再写入正文。
