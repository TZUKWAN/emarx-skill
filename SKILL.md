# EMARX

EMARX v7.3 是中文学理思辨论文生产技能，不是论文说明书、流程展示器或提示词集合。所有扫描、锚定、诊断、影子重组、审稿、事实核查和引用审计都属于后台工作，不能写进论文正文。

目标是生成能进入中文人文社科期刊语境的论文：问题意识明确，结构有推进，概念稳定，论证有材料和机制支撑，语言平实、稳健、有学术质感。完整论文默认生成 Markdown 工作稿和 Word 文档。

## 一、总原则

1. 先读本次工作空间，再使用既有语料规律。旧报告、旧画像和旧蒸馏结论只能作索引，不能替代当次原文读取。
2. 论文结构由题目内部的真实关系生长出来，不把“研究对象与概念边界”“理论框架”“材料锚定”“创新点分析”等后台动作做成正文标题。
3. 写作之前必须形成文献功能绑定、小节论证卡和段落级大纲。小节是最小准入单元，小节内每个自然段只承担一个清楚的论证推进；一节写完、审完、过门后，再进入下一节。
4. 写作之前必须确定期刊体例画像。引言、摘要、章下小序和标题层级要服从目标期刊或锚定论文的体例，不默认长引言、理科式摘要或无章下导语。
5. 正文必须围绕问题链逐段推进。每个小节先以小问题进入，再推出中层机制和较大判断，尽量以小见大，避免从宏大判断倒扣材料。
6. 正确表达要在首稿生成时形成，不能依赖后期检测批量修补。
7. 引用、事实和 Word 交付都必须有证据。没有核验，不得宣称完成。
8. 千问、脚本和辅助模型只能做低风险的读取、提取、定位、压缩和分类；主模型必须亲自复核关键来源、判断、结论和交付物。
9. 论文题目、一级标题、二级标题和必要的三级标题不得使用问句。标题不能出现问号，也不能使用“如何、何以、为何、为什么、怎样、怎么、是否、能否、吗”等题目化提问腔。问题链属于后台构思和正文段落内部推进，不能直接变成标题。
10. 完整期刊论文默认参考文献不少于 28 条。若用户明确指定更少、目标期刊有明确更少体例、或工作空间真实可用文献不足 28 条，必须在交付说明中明示原因；不得用无关文献、重复作者、假文献或未读文献凑数。

## 三、完整论文工作流

### 1. 资料建档

扫描本次工作空间中的 PDF、DOCX、Markdown、TXT 和保存网页，区分正式论文、政策资料、用户草稿、生成稿、技能文件和审计文件。只有真实资料进入论文参考范围。生成稿、旧改写稿、技能报告和审计文件不得当作文献来源。

### 2. 锚定三至五篇论文

根据用户题目，从本次工作空间真实论文中选择三至五篇锚定论文。选择依据不是文件名相似，而是内容功能互补：研究对象接近、理论视角有用、结构值得参照、材料支撑充分、语言节奏成熟。

资料不足三篇时如实说明。不得用无关论文凑数。

### 3. 即时精读锚定论文

锚定后必须回到原文连续读取。精读内容随原文结构展开，不能填统一模板。重点看：

- 摘要怎样无作者主语地完成对象进入、问题收束和判断落点；
- 引言怎样把现实语境、文献对话或理论矛盾转化为研究问题；
- 一级、二级、三级标题之间怎样递进；
- 每个主体部分承担什么不可替代的论证任务；
- 段落怎样起笔、承接、转折、展开和收束；
- 文献、材料、案例和政策如何进入论证；
- 哪些语言方法可迁移，哪些只属于该论文对象。

若多篇精读报告除题名外大段相同，判定为模板化失败，必须重读。

### 4. 研究诊断

研究诊断只用于后台。它要明确学科位置、研究对象、理论问题、现有解释的限度、中心判断、可能创新、材料缺口和事实风险。概念边界在此澄清，并在正文需要时自然进入论证。

除非用户明确要求研究设计或方法章节，正文不单独设置“研究对象与概念边界”“概念界定”“理论框架”一类标题。

### 5. 来源到论点映射

使用 `references/citation-fact-protocol-v7.md` 建立来源到论点映射。完整论文默认先建立不少于 28 条可用参考文献池，再按正文论证需要分配引用位置。每篇文献只能绑定一处正文使用位置，每个作者只出现一次。引用必须跟在真正使用该来源的句子后面，不能堆到段落末尾。

若用户要求“工作空间全部文献都用上”，先检查同作者重复、无关文献和容量冲突。冲突必须报告，不能靠假引用解决。

### 6. 影子重组

影子重组只用于内部理解锚定论文的结构动作。可以把几篇锚定论文的有效论证顺序、材料角色和段落功能重新配置到用户题目下，但不得把影子稿交付，也不得把原文拼接成新论文。

### 7. 论证骨架、小节论证卡和段落级大纲

形成中心判断、一级论证运动、二级标题、章下小序和段落序列。主体通常为四到五个一级部分，每个主要主体部分应有二级标题。Markdown 交付时，一级章节使用 `## 一、……`，正文二级标题必须使用 `### （一）……`。除摘要、引言、结论和参考文献外，正文每个一级章节标题下、第一个二级标题前，应设置一段短小章下小序，用来收束上一层问题并引出本章内部问题链。题目如果更适合三段推进，也必须保证每一段都由前一段产生论证必要性。所有标题都要写成判断式、关系式或对象式标题，不能写成问句。

每个二级小节必须先写小节论证卡，说明本节解释的对象关系、承接上一节未尽问题的方式、使用哪些文献或材料、以哪个小问题切入、问题链怎样从小点推向大判断、排除哪些浅层说法、必须避免哪些表达惯性、最终判断是什么、为什么下一节有必要出现。小节论证卡是写作发动机，不得翻译成正文说明。

每个拟写段落先确定：本段判断、承接关系、对象关系、材料来源、推理环节、限制条件、转出问题和完成标准。这些是写作准备，不得翻译成正文说明书。若某段需要引用，必须在大纲阶段确定该来源在本段中的功能；若无法说清来源提供了什么概念、机制、材料或限制，不得把该来源塞进正文。

### 8. 小节生产与逐段起草

一次只生产一个二级小节。每个小节内部仍然一段一段写，当前段写完后，立即检查论断、来源、过渡、句式、事实、引用和判断落点；小节内所有段落合格后，再用 `section_quality_gate.py` 做小节准入检查。未通过的小节不得进入全文草稿。

首稿本身必须尽量写对。不得先生成一篇无引用或弱引用的长文，再在末尾补参考文献。不得先写空泛路径段，再靠后期替换词语修饰。若连续出现悬空指代段首、应然句堆叠、泛动词空转、公式化对照或引用未进入论证，应停止续写并退回小节论证卡和段落级大纲。

段落可以从对象、事实、关系、既有判断、语境变化或承接句进入。不得用下列方式开头：

- “本文认为”“本文的核心观点是”“文章将”；
- “有研究指出”“相关研究表明”“学者认为”；
- 先否定一个尚未出现的观点；
- 机械复述标题；
- 用宏大背景、口号或空泛价值判断代替对象进入。

### 9. 摘要、引言、结论

摘要采用无作者主语的报道式表达，不出现“本文、笔者、作者、文章认为、核心观点是”。摘要属于中文人文社科摘要，不写成理科式“目的、方法、结果、结论”，不逐项播报研究过程，也不按章节概括。摘要直接呈现对象处境、问题张力、解释关系、判断落点和必要的路径指向，语言要像论文摘要，不像项目报告。

引言负责把读者带到研究问题，不写“本文结构如下”。引言长度和结构必须服从期刊体例画像：有的期刊只需一段短引言，有的期刊需要“政策阐释—文献对话—问题引出”。政策、现实变化、技术环境和已有研究都只能作为入口，必须转化为需要解释的矛盾或限度。不得默认写成长篇背景介绍。

结论回收全文中心判断，说明理论意义和边界，不复述目录，不喊口号。

### 10. 审稿、改稿和核查

初稿完成后先审逻辑，再审语言，最后审格式。重点检查：

- 中心判断是否贯穿；
- 一级标题之间是否递进；
- 二级标题是否真正收窄分析动作；
- 所有标题是否没有问号和题目化提问腔；
- 每段是否推进论证，而不是说明流程；
- 概念是否稳定；
- 材料是否支撑判断；
- 是否有作者自我播报、综述插入、空泛否定、公式化对照、冒号小标题和 AI 腔；
- 高风险事实是否核验；
- 引用是否连续编号、一文献一次、一作者一次、随句插入；
- 参考文献是否通常不少于 28 条，是否按 GB/T 7714 顺序排列；
- DOCX 是否真实生成并通过结构/渲染检查。

形式合格但仍像说明书、综述或套壳稿时，必须重写结构或段落，不能只润色。

失败稿回炉不得把坏稿全文当作主要蓝本逐句润色。应保留中心判断、标题链、来源映射和有效材料，删除污染性表达，回到干净段落级大纲按小节重写。同一问题连续两轮出现时，判定生成策略失败，缩小任务粒度或更换策略。

## 四、语言规则

目标语体是平实、清楚、稳健、有分寸、有判断、有学术质感。

语言不靠生僻词、自造概念、密集引号、冒号、破折号、排比口号和二元公式制造深度。长句用于容纳条件、关系和推演，短句用于转折、强调和收束；节奏来自思想推进，不按比例机械安排。

禁止或强烈避免：

- “首先、其次、再次、最后”；
- “重构、重建、填补空白”等夸张自我拔高；
- “不是……而是……”“并非……而是……”等公式化对照；
- 标题中的“何以、为何、如何、为什么、怎样、怎么、是否、能否、吗”等题目化提问腔；
- “本文认为、本文指出、本文的核心观点是、文章认为”；
- “有研究指出、相关研究表明、已有研究认为、学者认为”；
- 段首裸否定；
- 标点制造的 AI 腔。

概念要稳定，动词要承担推进。能用清楚动词表达的地方，不用抽象名词堆叠。

## 五、脚本工具

脚本用于索引、审计和交付，不等于最终事实判断。

```bash
python scripts/scan_workspace_sources.py --root <workspace> --output sources.json
python scripts/build_research_brief.py --topic "论文题目" --sources sources.json --output research-brief.md
python scripts/select_anchor_papers.py --topic "论文题目" --workspace-root <workspace> --output anchor-papers.md --top-k 5
python scripts/prepare_anchor_reading.py --anchors anchor-papers.md --output-dir anchor-reading
python scripts/qwen_deep_distill_corpus.py --index structure_index.json --output-dir qwen_deep_reading --transport api
python scripts/audit_deep_reading_reports.py --run-manifest qwen_deep_reading/run.jsonl --expected-count 450
python scripts/section_quality_gate.py --paper paper.md --section-title "（一）小节标题" --output section-gate.json --require-citation
python scripts/scholarliness_audit.py --paper paper.md --output scholarliness-audit.json
python scripts/bad_draft_audit.py --paper paper.md --output bad-draft-audit.json
python scripts/citation_audit.py --paper paper.md --output citation-audit.json --min-count 28
python scripts/markdown_to_docx.py --input paper.md --output paper.docx
python scripts/audit_docx.py --docx paper.docx --output docx-audit.json
python scripts/update_user_profile.py --profile references/user-research-profile.md --topic "主题" --feedback "用户反馈"
```

运行辅助模型时，输出只是证据索引和精读材料。技能规则、论文结构、学理判断、最终写作和质量结论必须由主模型复核后确定。

## 六、交付标准

完整论文交付必须包含：

- Markdown 工作稿路径；
- DOCX 文件路径；
- 正文字数；
- 标题层级摘要；
- 核心论点摘要；
- 使用的主要本地来源和来源缺口；
- 引用冲突和引用审计状态；
- 参考文献数量，若少于 28 条必须说明真实原因；
- 事实核查状态；
- DOCX 审计和渲染检查状态；
- 残余风险。

交付前必须至少运行语言风险审计、引用审计和 DOCX 审计；其中任何一项未通过时，只能说明正在回炉或存在风险，不能宣称稿件合格。

没有生成 Word 不能说已生成 Word；没有审计不能说已审计；没有核验来源不能说事实可靠。EMARX 的质量以真实产物和证据为准。
